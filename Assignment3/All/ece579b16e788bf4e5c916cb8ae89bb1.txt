			<h1 id="section_0">Event chain methodology</h1><ul id="page-actions" class="hlist"><li id="ca-edit"></li><li id="ca-upload"></li><li id="ca-talk"><a href="/wiki/Talk:Event_chain_methodology" title="Discussion about the content page [t]" accesskey="t">Talk</a></li><li id="ca-watch" class="watch-this-article"><a href="https://en.m.wikipedia.org/w/index.php?title=Special:UserLogin&amp;returnto=Event+chain+methodology" title="Add this page to your watchlist [w]" accesskey="w"></a></li></ul>			<div id="content" class="content">
				<div><div class="thumb tright">
<div class="thumbinner" style="width:322px;"><a href="/wiki/File:EventChain.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/f/f9/EventChain.jpg/320px-EventChain.jpg" width="320" height="232" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/f/f9/EventChain.jpg/480px-EventChain.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/f/f9/EventChain.jpg/640px-EventChain.jpg 2x"></a>
<div class="thumbcaption">

Event chain diagram</div>
</div>
</div>
<p><b>Event chain methodology</b> is an uncertainty modeling and schedule <a href="/wiki/Network_theory" title="Network theory">network analysis</a> technique that is focused on identifying and managing events and event chains that affect <a href="/wiki/Project" title="Project">project</a> schedules. Event chain methodology is the next advance beyond <a href="/wiki/Critical_path_method" title="Critical path method">critical path method</a> and <a href="/wiki/Critical_chain" title="Critical chain" class="mw-redirect">critical chain</a> project management.<sup id="cite_ref-1" class="reference"><a href="#cite_note-1"><span>[</span>1<span>]</span></a></sup></p>
<p>Event chain methodology helps to mitigate the effect of <a href="/wiki/Motivation" title="Motivation">motivational</a> and <a href="/wiki/Cognitive_biases" title="Cognitive biases" class="mw-redirect">cognitive biases</a> in <a href="/wiki/Estimation" title="Estimation">estimating</a> and <a href="/wiki/Scheduling_(production_processes)" title="Scheduling (production processes)">scheduling</a>.<sup id="cite_ref-2" class="reference"><a href="#cite_note-2"><span>[</span>2<span>]</span></a></sup><sup id="cite_ref-3" class="reference"><a href="#cite_note-3"><span>[</span>3<span>]</span></a></sup></p>
<p><br></p>

</div><h2><span class="mw-headline" id="Event_Chain_Methodology_Principles">Event Chain Methodology Principles</span><a href="#editor/1" data-section="1" class="edit-page">Edit</a></h2><div>
<h3><span class="mw-headline" id="Moment_of_risk_and_state_of_activity">Moment of risk and state of activity</span><a href="#editor/2" data-section="2" class="edit-page">Edit</a></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:322px;"><a href="/wiki/File:EventChainMethodology.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/e/ed/EventChainMethodology.jpg/320px-EventChainMethodology.jpg" width="320" height="141" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/e/ed/EventChainMethodology.jpg/480px-EventChainMethodology.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/e/ed/EventChainMethodology.jpg/640px-EventChainMethodology.jpg 2x"></a>
<div class="thumbcaption">

Moment of risk and state of activity</div>
</div>
</div>
<p>An activity (task) in most real life processes is not a continuous uniform procedure. Tasks are affected by external events, which transform an activity from one state to another.</p>
<p>One of the important properties of an event is the moment when an event occurs during the course of an activity. This moment, when an event occurs, in most cases is probabilistic and can be defined using <a href="/wiki/Statistical_distribution" title="Statistical distribution" class="mw-redirect">statistical distribution</a>.</p>
<h3><span class="mw-headline" id="Event_Chains">Event Chains</span><a href="#editor/3" data-section="3" class="edit-page">Edit</a></h3>
<p>Events can cause other events, which will create event chains. These event chains can significantly affect the course of the project. For example, requirement changes can cause an activity to be delayed. To accelerate the activity, the project manager allocates a resource from another activity, which then leads to a missed deadline. Eventually, this can lead to the failure of the project.</p>
<h3><span class="mw-headline" id="Monte_Carlo_Simulations">Monte Carlo Simulations</span><a href="#editor/4" data-section="4" class="edit-page">Edit</a></h3>
<p>Once events and event chains are defined, quantitative analysis using <a href="/wiki/Monte_Carlo_simulation" title="Monte Carlo simulation" class="mw-redirect">Monte Carlo simulation</a> can be performed to quantify the cumulative effect of the events. Probabilities and effects of risks are used as input data for Monte Carlo simulation of the project schedule.<sup id="cite_ref-4" class="reference"><a href="#cite_note-4"><span>[</span>4<span>]</span></a></sup> In most real life projects, it is necessary to supplement the information regarding the uncertainties expressed as an event, with distributions related to duration, start time, cost, and other parameters.</p>
<h3><span class="mw-headline" id="Critical_Event_Chains">Critical Event Chains</span><a href="#editor/5" data-section="5" class="edit-page">Edit</a></h3>
<p>The single events or the event chains that have the most potential to affect the projects are the “critical events” or “critical chains of events.” By identifying critical events or critical chains of events, we can mitigate their negative effects. These critical chains of events can be identified by analyzing the correlations between the main project parameters, such as project duration or cost, and the event chains.</p>
<h3><span class="mw-headline" id="Performance_Tracking_with_Event_Chains">Performance Tracking with Event Chains</span><a href="#editor/6" data-section="6" class="edit-page">Edit</a></h3>
<p>Monitoring the activity's progress ensures that updated information is used to perform the analysis. During the course of the project, the probability and time of the events can be recalculated based on actual data. The main issue with performance tracking is forecasting an activity’s duration and cost if an activity is partially completed and certain events are assigned to the activity. The simple heuristic approach to this problem is to analyze the moment of risk, which is defined as one of the event parameters. Advanced analysis can be performed using a Bayesian approach.</p>
<h3><span class="mw-headline" id="Event_Chain_Diagrams">Event Chain Diagrams</span><a href="#editor/7" data-section="7" class="edit-page">Edit</a></h3>
<p>Event Chain Diagrams are visualizations that show the relationships between events and tasks and how the events affect each other. The simplest way to represent these chains is to depict them as arrows associated with certain tasks or time intervals on the Gantt chart. Different events and event chains can be displayed using different colors. Events can be global (for all tasks in the project) and local (for a particular task). By using Event Chain Diagrams to visualize events and event chains, the modeling and analysis of risks and uncertainties can be significantly simplified.</p>
</div><h2><span class="mw-headline" id="Event_Chain_Methodology_Phenomena">Event Chain Methodology Phenomena</span><a href="#editor/8" data-section="8" class="edit-page">Edit</a></h2><div>
<h3><span class="mw-headline" id="Repeated_Activities">Repeated Activities</span><a href="#editor/9" data-section="9" class="edit-page">Edit</a></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:322px;"><a href="/wiki/File:RepeatedActivity.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/c/c5/RepeatedActivity.jpg/320px-RepeatedActivity.jpg" width="320" height="110" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/c/c5/RepeatedActivity.jpg/480px-RepeatedActivity.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/c/c5/RepeatedActivity.jpg/640px-RepeatedActivity.jpg 2x"></a>
<div class="thumbcaption">

Repeated Acitivity</div>
</div>
</div>
<p>Sometimes events can cause the start of an activity that has already been completed. This is a very common scenario for real life projects; sometimes a previous activity must be repeated based on the results of a succeeding activity. Modeling of these scenarios using event chain methodology is simple. The original project schedule does not need to be updated, as all that is required is to define the event and assign it to an activity that points to the previous activity. In addition, a limit to the number of times an activity can be repeated needs to be defined.</p>
<h3><span class="mw-headline" id="Event_Chains_and_Risk_Mitigation">Event Chains and Risk Mitigation</span><a href="#editor/10" data-section="10" class="edit-page">Edit</a></h3>
<div class="thumb tright">
<div class="thumbinner" style="width:252px;"><a href="/wiki/File:MitigationPlan.jpg" class="image"><img alt="" src="//upload.wikimedia.org/wikipedia/commons/thumb/0/00/MitigationPlan.jpg/250px-MitigationPlan.jpg" width="250" height="266" class="thumbimage" srcset="//upload.wikimedia.org/wikipedia/commons/thumb/0/00/MitigationPlan.jpg/375px-MitigationPlan.jpg 1.5x, //upload.wikimedia.org/wikipedia/commons/thumb/0/00/MitigationPlan.jpg/500px-MitigationPlan.jpg 2x"></a>
<div class="thumbcaption">

Mitigation plan</div>
</div>
</div>
<p>If event or event chain occurs during the course of a project, it may require some mitigation effort. In some cases, mitigation plans can be generated. Mitigation plans are an activity or group of activities (small schedule) that augment the project schedule if a certain event occurs. The solution is to assign the mitigation plan to an event or event chain. These small schedules will be triggered when an event chain occurs. The same mitigation plan can be used for different events.</p>
<h3><span class="mw-headline" id="Resource_Allocation_Based_on_Events">Resource Allocation Based on Events</span><a href="#editor/11" data-section="11" class="edit-page">Edit</a></h3>
<p>One potential event is the reassignment of a resource from one activity to another, which can occur under certain conditions. For example, if an activity requires more resources to complete it within a fixed period, this will trigger an event to reallocate the resource from another activity. Reallocation of resources can also occur when activity duration reaches a certain deadline or the cost exceeds a certain value. Events can be used to model different situations with resources, e.g. temporary leave, illness, vacations, etc.</p>
