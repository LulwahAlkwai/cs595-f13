		<h1 class="entry-title">Visualizing neural networks from the nnet&nbsp;package</h1>
	</header><!-- .entry-header -->

	<div class="entry-content">
		<p>Neural networks have received a lot of attention for their abilities to &#8216;learn&#8217; relationships among variables.  They represent an innovative technique for model fitting that doesn&#8217;t rely on conventional assumptions necessary for standard models and they can also quite effectively handle multivariate response data.  A neural network model is very similar to a non-linear regression model, with the exception that the former can handle an incredibly large amount of model parameters.  For this reason, neural network models are said to have the ability to approximate any continuous function. </p>
<p>I&#8217;ve been dabbling with neural network models for my &#8216;research&#8217; over the last few months.  I&#8217;ll admit that I was drawn to the approach given the incredible amount of hype and statistical voodoo that is attributed to these models.  After working with neural networks, I&#8217;ve found that they are often no better, and in some cases much worse, than standard statistical techniques for predicting relationships among variables.  Regardless, the foundational theory of neural networks is pretty interesting, especially when you consider how computer science and informatics has improved our ability to create useful models.  </p>
<p>R has a few packages for creating neural network models (<a href="http://cran.r-project.org/web/packages/neuralnet/index.html">neuralnet</a>, <a href="http://cran.r-project.org/web/packages/nnet/index.html">nnet</a>, <a href="http://cran.r-project.org/web/packages/RSNNS/index.html">RSNNS</a>).  I have worked extensively with the nnet package created by Brian Ripley.  The functions in this package allow you to develop and validate the most common type of neural network model, i.e, the feed-forward multi-layer perceptron.  The functions have enough flexibility to allow the user to develop the best or most optimal models by varying parameters during the training process.  One major disadvantage is an inability to visualize the models.  In fact, neural networks are commonly criticized as &#8216;black-boxes&#8217; that offer little insight into causative relationships among variables.  Recent research, primarily in the ecological literature, has addressed this criticism and several approaches have since been developed to &#8216;illuminate the black-box&#8217;.<a href='#ref1'><sup>1</sup></a>    </p>
<p>As far as I know, none of the recent techniques for evaluating neural network models are available in R.  &Ouml;zesmi and &Ouml;zemi (1999)<a href='#ref2'><sup>2</sup></a> describe a neural interpretation diagram (NID) to visualize parameters in a trained neural network model.  These diagrams allow the modeler to qualitatively examine the importance of explanatory variables given their relative influence on response variables, using model weights as inference into causation.  More specifically, the diagrams illustrate connections between layers with width and color in proportion to magnitude and direction of each weight.  More influential variables would have lots of thick connections between the layers.  </p>
<p>In this blog I present a function for plotting neural networks from the nnet package.  This function allows the user to plot the network as a neural interpretation diagram, with the option to plot without color-coding or shading of weights.  The neuralnet package also offers a plot method for neural network objects and I encourage interested readers to <a href="http://gekkoquant.com/2012/05/26/neural-networks-with-r-simple-example/">check it out</a>.  I have created the function for the nnet package given my own preferences for aesthetics, so its up to the reader to choose which function to use.  I plan on preparing this function as a contributed package to CRAN, but thought I&#8217;d present it in my blog first to gauge interest. </p>
<p>Let&#8217;s start by creating an artificial dataset to build the model.  This is a similar approach that I used in my previous blog about <a href="http://beckmw.wordpress.com/2013/02/05/collinearity-and-stepwise-vif-selection/">collinearity</a>.  We create eight random variables with an arbitrary correlation struction and then create a response variable as a linear combination of the eight random variables.</p>
<pre class="brush: r; title: ; notranslate" title="">
require(clusterGeneration)

set.seed(2)
num.vars&lt;-8
num.obs&lt;-1000

#arbitrary correlation matrix and random variables
cov.mat&lt;-genPositiveDefMat(num.vars,covMethod=c(&quot;unifcorrmat&quot;))$Sigma
rand.vars&lt;-mvrnorm(num.obs,rep(0,num.vars),Sigma=cov.mat)
parms&lt;-runif(num.vars,-10,10)

#response variable as linear combination of random variables and random error term
y&lt;-rand.vars %*% matrix(parms) + rnorm(num.obs,sd=20)
</pre>
<p>Now we can create a neural network model using our synthetic dataset.  The <code>nnet</code> function can input a formula or two separate arguments for the response and explanatory variables (we use the latter here).  We also have to convert the response variable to a 0-1 continuous scale in order to use the <code>nnet</code> function properly (via the <code>linout</code> argument, see the <a href="http://cran.r-project.org/web/packages/nnet/nnet.pdf">documentation</a>).  </p>
<pre class="brush: r; title: ; notranslate" title="">
require(nnet)

rand.vars&lt;-data.frame(rand.vars)
y&lt;-data.frame((y-min(y))/(max(y)-min(y)))
names(y)&lt;-'y'

mod1&lt;-nnet(rand.vars,y,size=10,linout=T)
</pre>
<p>We&#8217;ve created a neural network model with ten nodes in the hidden layer and a linear transfer function for the response variable.  All other arguments are as default.  The tricky part of developing an optimal neural network model is identifying a combination of parameters that produces model predictions closest to observed.  Keeping all other arguments as default is not a wise choice but is a trivial matter for this blog.  Next, we use the plot function now that we have a neural network object.</p>
<p>First we import the function from my Github account (aside: does anyone know a better way to do this?).</p>
<pre class="brush: r; title: ; notranslate" title="">
#import function from Github
require(RCurl)

root.url&lt;-'https://gist.github.com/fawda123'
raw.fun&lt;-paste(
  root.url,
  '5086859/raw/17fd6d2adec4dbcf5ce750cbd1f3e0f4be9d8b19/nnet_plot_fun.r',
  sep='/'
  )
script&lt;-getURL(raw.fun, ssl.verifypeer = FALSE)
eval(parse(text = script))
rm('script','raw.fun')
</pre>
<p>The function is now loaded in our workspace as <code>plot.nnet</code>.  We can use the function just by calling <code>plot</code> since it recognizes the neural network object as input.  </p>
<pre class="brush: r; title: ; notranslate" title="">
par(mar=numeric(4),mfrow=c(1,2),family='serif')
plot(mod1,nid=F)
plot(mod1)
</pre>
<p><a href="http://beckmw.files.wordpress.com/2013/03/nnet1.pdf"><img src="http://beckmw.files.wordpress.com/2013/03/nnet1.jpg?w=525" /></a></p>
<p>The image on the left is a standard illustration of a neural network model and the image on the right is the same model illustrated as a neural interpretation diagram (default plot).  The black lines are positive weights and the grey lines are negative weights.  Line thickness is in proportion to magnitude of the weight relative to all others.  Each of the eight random variables are shown in the first layer (labelled as X1-X8) and the response variable is shown in the far right layer (labelled as &#8216;y&#8217;).  The hidden layer is labelled as H1 through H10, which was specified using the <code>size</code> argument in the <code>nnet</code> function.  B1 and B2 are bias layers that apply constant values to the nodes, similar to intercept terms in a regression model.  </p>

<td>numeric value (0-1) indicating transparency of connections, default 1</td>
</tr>
<tr>
<td><code>circle.col</code></td>
<td>text value indicating color of nodes, default &#8216;lightgrey&#8217;</td>
</tr>
<tr>
<td><code>pos.col</code></td>
<td>text value indicating color of positive connection weights, default &#8216;black&#8217;</td>
</tr>
<tr>
<td><code>neg.col</code></td>
<td>text value indicating color of negative connection weights, default &#8216;grey&#8217;</td>
</tr>
<tr>
<td><code>...</code></td>
<td>additional arguments passed to generic plot function</td>
</tr>
</table>
<p>Most of the arguments can be tweaked for aesthetics.  We&#8217;ll illustrate using a neural network model created in the example code for the <code>nnet</code> function:</p>
<pre class="brush: r; title: ; notranslate" title="">
#example data and code from nnet function examples
ir&lt;-rbind(iris3[,,1],iris3[,,2],iris3[,,3])
targets&lt;-class.ind( c(rep(&quot;s&quot;, 50), rep(&quot;c&quot;, 50), rep(&quot;v&quot;, 50)) )
samp&lt;-c(sample(1:50,25), sample(51:100,25), sample(101:150,25))
ir1&lt;-nnet(ir[samp,], targets[samp,], size = 2, rang = 0.1,decay = 5e-4, maxit = 200)

#plot the model with different default values for the arguments
par(mar=numeric(4),family='serif')
plot.nnet(ir1,pos.col='darkgreen',neg.col='darkblue',alpha.val=0.7,rel.rsc=15,
circle.cex=10,cex=1.4,
	circle.col='brown')
</pre>
<p><a href="http://beckmw.files.wordpress.com/2013/03/nnet2.pdf"><img src="http://beckmw.files.wordpress.com/2013/03/nnet2.jpg?w=525" /></a></p>
<p>The neural network plotted above shows how we can tweak the arguments based on our preferences.  This figure also shows that the function can plot neural networks with multiple response variables (&#8216;c&#8217;, &#8216;s&#8217;, and &#8216;v&#8217; in the iris dataset).  </p>
<p>Another useful feature of the function is the ability to get the connection weights from the original <code>nnet</code> object.  Admittedly, the weights are an attribute of the original function but they are not nicely arranged.  We can get the weight values directly with the <code>plot.nnet</code> function using the <code>wts.only</code> argument.</p>
<pre class="brush: r; title: ; notranslate" title="">
plot.nnet(ir1,wts.only=T)

$`hidden 1`
[1]  0.2440625  0.5161636  1.9179850 -2.8496175
[5] -1.4606777

$`hidden 2`
[1]   9.222086   6.350143   7.896035 -11.666666
[5]  -8.531172

$`out 1`
[1]  -5.868639 -10.334504  11.879805

$`out 2`
[1] -4.6083813  8.8040909  0.1754799

$`out 3`
[1]   6.2251557  -0.3604812 -12.7215625
</pre>
<p>The function returns a list with all connections to the hidden layer (hidden 1 through hidden 2) and all connections to the output layer (out1 through out3).  The first value in each element of the list is the weight from the bias layer.  </p>
<p>The last features of the <code>plot.nnet</code> function we&#8217;ll look at are the <code>all.in</code> and <code>all.out</code> arguments.  We can use these arguments to plot connections for specific variables of interest.  For example, what if we want to examine the weights that are relevant only for sepal width (&#8216;Sepal W.&#8217;) and virginica sp. (&#8216;v&#8217;)? </p>
<pre class="brush: r; title: ; notranslate" title="">
plot.nnet(ir1,pos.col='darkgreen',neg.col='darkblue',alpha.val=0.7,rel.rsc=15,
	circle.cex=10,cex=1.4,circle.col='brown',all.in='Sepal W.',all.out='v')
</pre>
<p><a href="http://beckmw.files.wordpress.com/2013/03/nnet3.pdf"><img src="http://beckmw.files.wordpress.com/2013/03/nnet3.jpg?w=525" /></a></p>
<p>This exercise is relatively trivial for a small neural network model but can be quite useful for a <a href="http://beckmw.files.wordpress.com/2013/03/nnet_ex.pdf">larger model</a>.  </p>
<p>Feel free to grab the function from Github (linked above).  I encourage suggestions on ways to improve its functionality.  I will likely present more quantitative methods of evaluating neural networks in a future blog, so stay tuned!  </p>
