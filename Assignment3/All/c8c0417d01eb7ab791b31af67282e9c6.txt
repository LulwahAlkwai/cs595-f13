 <h2><a name="msinttypes"></a>msinttypes<a href="#msinttypes" class="section_anchor"></a></h2><p>This project fills the absence of <tt>stdint.h</tt> and <tt>inttypes.h</tt> in Microsoft Visual Studio. This files were standartized by ISO/IEC as a part of <a href="http://www.open-std.org/jtc1/sc22/wg14/www/docs/n1124.pdf" rel="nofollow">C99</a> standard library. If you want to compile or use C99 compliant project with Microsoft Visual Studio, you will likely find that you&#x27;re missing these headers. Note though, that just adding these header does not make Visual Studio compiler fully C99 compliant. </p><p><a href="http://ffmpeg.mplayerhq.hu/" rel="nofollow">FFMpeg</a> is a good example of library that requires <tt>inttypes.h</tt> even if you just want to link your program with it. </p><p>More portable version of <tt>stdint.h</tt> can be find <a href="http://www.azillionmonkeys.com/qed/pstdint.h" rel="nofollow">here</a> but it does not implement <tt>inttypes.h</tt> features, such as printf and scanf format specifiers. </p><h2><a name="Changes_history"></a>Changes history<a href="#Changes_history" class="section_anchor"></a></h2><pre class="prettyprint">------------------------------------------------------------------------
